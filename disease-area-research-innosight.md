# 疾病领域调研（InnoSight 专用完整版）

## 目标

以医药行业高级分析师的标准，优先依靠 InnoSight 的结构化医药数据建立完整母表，再用监管、试验注册、指南、论文和交易双方公告核验关键结论。覆盖率、可追溯性和决策价值优先于响应速度与篇幅节省。

## 不可妥协的运行条件

- 必须检测到 `innosight_mcp`，并首先调用 `mcp__innosight_mcp__list_data_apis`。
- 调用任何数据 API 前，必须以目标 API path 调用 `mcp__innosight_mcp__get_data_api_schema`；不得猜测字段、枚举或分页参数。
- 优先调用专用数据接口；只有专用接口不适用时才使用 `mcp__innosight_mcp__query_data_api`。
- 对管线、临床试验、交易、新闻等集合型结果完整翻页，不得只取第一页或任意截断代表性结果。
- InnoSight 可用时，不得先以网页搜索代替数据库母表；外部来源用于补缺和一手核验。
- 若 InnoSight 未启用、无权限或核心接口持续失败，停止假装执行完整版，明确说明阻塞，并建议改用 `disease-area-research` 通用版。

## 输入与默认范围

- 将疾病名称作为唯一必填输入。若用户未提供，只追问具体疾病领域。
- 默认覆盖全球与中国、创新治疗药物及主要亚型；数据截止日为执行当天。
- 默认回溯最近 24 个月 BD 与并购。
- 对宽泛疾病自动建立亚型层级和纳入/排除边界；除非疾病本身无法识别，否则不要求用户重复指定报告结构。
- 默认使用中文和专业、客观、审慎的医药行业分析口吻。

## 执行前必须读取

完整读取以下文件后再调用数据：

1. `references/source-and-research-protocol.md`：精确工具顺序、接口覆盖、分页、外部核验与冲突处理。
2. `references/golden-schema.md`：完整五部分报告结构、字段和分析要求。
3. `references/quality-checklist.md`：交付前覆盖率与证据检查。

## 执行流程

### 0. InnoSight 预检

1. 调用 `mcp__innosight_mcp__list_data_apis`，保存可用接口清单。
2. 将接口清单映射到流行病学、指南/治疗格局、已获批药物、全球管线、临床试验设计与时间线、试验结果、中国注册、BD交易、新闻和专利背景。
3. 对计划使用的每个接口读取 schema，记录 API path、准确参数、分页方式和返回字段。
4. 若关键数据域不存在，先记录缺口，再规划官方外部来源补缺；不得直接跳过。

### 1. 疾病范围与标准疗法

1. 规范中文名、英文名、缩写、历史名称、主要亚型和检索同义词。
2. 获取全球与中国流行病学、疾病负担、最新指南和现行 SoC。
3. 从患者结局、疗效、安全性、复发/耐药、特定人群、诊断、给药、可及性与成本形成未满足需求清单。

### 2. 全球管线母表

1. 使用疾病总称、同义词、旧称和关键亚型分别检索并完整翻页。
2. 按资产别名、盐型、合作方和地区权益归一去重。
3. 对全部活跃资产记录靶点/机制、modality、企业、全球最高阶段、中国阶段、目标人群、试验和最新状态。
4. 按靶点/机制和 modality 双维度统计阶段、拥挤度和主要企业。
5. 对已上市、关键/注册性试验、Phase 2/3、重要读出或机制差异资产继续查询设计、时间线和结果。

### 3. 中国本土布局

1. 从全球母表中识别总部或主要创新研发主体位于中国的企业。
2. 区分自主发现、License-in、共同开发和区域权益。
3. 核对临床前、IND、Phase 1/2/3、NDA/BLA、获批上市和商业化状态。
4. 分析同质化、潜在 FIC/BIC、本地可及性、数据质量、开发、IP 和商业化风险。

### 4. BD 与并购

1. 系统检索截止日前 24 个月的许可、合作研发、共同商业化、选择权、资产收购和公司并购。
2. 对疾病总称、主要亚型、核心靶点和重点资产补检。
3. 记录日期、双方、标的、机制、权益地区、首付款、股权/近期付款、里程碑、潜在总额、royalty 和状态。
4. 未披露写“未披露”；不得重复加总或将 `up to` 写成确定付款。

### 5. 外部一手核验

对关键数字、阶段、审批、临床结果、指南和交易金额，使用 FDA、NMPA/CDE、EMA、ClinicalTrials.gov、ChiCTR、论文、会议、监管申报或交易双方公告核验。记录发布日期、数据截止日和访问日期。

### 6. 撰写与质控

1. 严格使用 Golden Schema 的五个一级部分，不增加平行的第六部分。
2. 重要管线、临床结果和交易使用结构化表格；表后给出判断而非条目堆积。
3. 区分已核验事实、跨来源推断、分析判断和未知事项。
4. 完成质量检查后，先输出简短“研究执行回执”，再输出正式报告。

## 研究执行回执

正式报告前必须给出紧凑回执，至少包含：

| 数据域 | InnoSight API/path | Schema 已读取 | 查询词/过滤条件 | 页数或 cursor 批次 | 返回/去重数量 | 外部核验 | 状态/缺口 |
|---|---|---|---|---:|---:|---|---|

回执规则：

- 不泄露账号凭证、内部 token 或敏感配置。
- 无法获得准确页数时，写明分页终止依据，例如 `has_more=false`、无 `next_cursor` 或已达到 `totalCount`。
- 任何核心数据域未完成时标记“未完成”，并在报告相应章节降低结论置信度。
- 不得用“已调用 InnoSight”一句话代替接口、分页和结果数量证明。

## 质量底线

- 不把搜索摘要、模型常识或新闻稿作为关键数字的唯一依据。
- 不把试验获批写成实际启动，不混淆全球阶段与中国阶段。
- 不把单臂早期数据写成确证性比较证据，不对不可比跨试验数字作简单排名。
- 不混用患病、确诊、治疗、可治疗和市场患者池。
- 不凭企业宣称判定 FIC/BIC。
- 不把专利背景检索表述为正式 FTO 或法律意见。
- 对检索不到、未披露、接口缺失或来源冲突的内容明确标注，不补写数字。
# 数据源与调研协议

## 目录

1. 数据源优先级
2. InnoSight 必须执行的调用流程
3. 分主题检索路径
4. 外部补缺策略
5. 证据记录与冲突处理

## 1. 数据源优先级

按以下顺序获取和核验信息：

1. InnoSight MCP 的结构化医药数据及其证据来源。
2. 一手权威来源：FDA、NMPA/CDE、EMA、WHO、国家卫健委、CDC、ClinicalTrials.gov、WHO ICTRP、ChiCTR、指南发布机构、公司监管申报文件和交易双方公告。
3. 同行评议论文、正式学术会议摘要/演示、疾病登记研究和高质量系统综述。
4. 可信行业媒体、公司新闻稿和投资者材料，用于补足最新进展；关键结论尽量交叉核验。
5. 市场研究报告仅用于市场规模参考，必须注明基准年、预测期、地域、口径及实际值/预测值。

不得以搜索结果摘要、未经核验的聚合页面或模型常识作为关键数字的唯一依据。

## 2. InnoSight 必须执行的调用流程

1. 首先调用 `mcp__innosight_mcp__list_data_apis`，确认当前账户可用接口。
2. 在调用任一数据查询接口之前，先以其 API path 调用 `mcp__innosight_mcp__get_data_api_schema`，读取准确的 `searchParams` 字段、枚举、返回结构和翻页方式。禁止猜测字段。
3. 优先调用专用接口；仅在专用接口不适用时使用 `mcp__innosight_mcp__query_data_api`。
4. 对需要完整覆盖的管线、临床试验、交易等结果持续翻页。依据 `pagination.has_more`、`next_cursor`、`totalCount`、`pageNum` 和 `pageSize` 判断；不得只取第一页。
5. 保存检索口径：疾病词、同义词、亚型、地区、阶段、交易日期范围和检索日期。

### 2.1 完整性门槛

- 至少尝试覆盖：流行病学、指南/治疗格局、已获批药物、全球管线、全球临床试验、临床设计/时间线、试验结果、中国注册、BD交易和近期新闻。
- 若竞争或 IP 判断是核心任务，补充 `search_global_pipeline_patent`；该结果仅作专利背景，不代替正式 FTO。
- 每个计划使用的接口均先读取 schema，并在执行回执中记录实际 API path。
- 集合型查询持续翻页，直到 `pagination.has_more=false`、`next_cursor` 为空、返回量达到 `totalCount`，或接口明确返回末页。
- 若同一接口按疾病总称无法覆盖主要亚型，使用同义词、旧称和亚型分别补检，再统一去重。
- 不得因为首批结果看起来“足够”而提前终止。

优先评估的专用接口包括：

- `search_epidemiology`：患病率、发病率、患者人数、疾病负担与趋势。
- `search_therapeutic_landscape_guide_recs`：指南推荐和治疗格局。
- `search_pipeline_approved_indication`：已获批药物及地区布局。
- `search_global_pipeline`：全球管线、靶点、药物类型、企业和阶段。
- `search_global_clinical`、`search_global_clinical_core_design`、`search_global_clinical_timeline`：试验状态、设计和时间线。
- `search_trial_result`：临床结果、终点、会议/文献证据和安全性。
- `search_domestic_registration`：中国注册与审评审批进度。
- `search_deals`：授权、合作、并购与交易金额。
- `search_pharmaceutical_news`：近期进展补充与时效性核查。
- `search_global_pipeline_patent`：仅在竞争判断需要时补充专利背景，不替代正式FTO分析。

## 3. 分主题检索路径

### 疾病背景与指南

- 同时检索疾病总称、主要亚型和旧称。
- 优先获取美国与中国最新版指南；必要时补充欧洲或国际共识。
- 记录指南名称、发布组织、版本/年份、患者分层、治疗线次和推荐强度。
- 流行病学分别报告全球与中国；区分患病率、发病率、诊断人数、治疗人数和预测人数。

### 全球管线与临床证据

- 先拉取疾病相关完整管线，再按靶点/机制、Modality、阶段、地区和企业去重。
- 同一资产的不同代号、盐型或合作方记录归一化，但保留地区权利和联合开发关系。
- 对代表性资产继续查询试验设计、关键时间线和结果。优先选择已上市、注册性/关键试验、Phase 2/3、有重要临床读出或具有显著机制差异的资产。
- 记录试验人群、样本量、分期、对照、随访、主要终点、关键疗效、安全性和数据成熟度。
- 分开报告全球最高阶段与中国阶段，不把“获准开展临床”写成“进入临床”。

### 中国本土布局

- 默认将中国本土企业定义为总部或主要创新研发主体位于中国的企业。
- 区分自主发现、License-in、共同开发和中国区域权益。
- 核对临床前、IND获批、Phase 1/2/3、NDA/BLA、获批上市和商业化状态。
- 差异化从机制、分子设计、给药方式、患者选择、疗效、安全性、便利性、成本和全球开发能力判断。

### BD交易与并购

- 检索区间为截止日前24个月，必要时纳入稍早但对当前格局有决定性影响的交易，并说明例外原因。
- 纳入与疾病治疗资产直接相关的 License-in/out、合作研发、共同商业化、选择权、资产收购和公司并购。
- 记录公告日期、交易双方、资产/平台、权益范围、地区、首付款、近期付款、研发/注册/销售里程碑、潜在总额、特许权使用费和交易状态。
- 保留公告原币种；只有在确有必要时换算，并注明汇率日期。严格区分“总对价”“潜在最高金额”和“未披露”。

## 4. 外部补缺策略

仅在 InnoSight 未覆盖、证据过旧或需要一手核验时使用外部来源：

- 指南：发布学会/协会官网及正式期刊版本。
- 流行病学：WHO、GBD、国家统计/卫生机构、疾病登记和高质量人群研究。
- 临床：试验注册、论文、会议官网、监管审评文件和申办方公告。
- 上市与审批：监管机构数据库和正式标签。
- BD：交易双方公告、监管申报、年报/季报和可信财经媒体。
- 市场规模：优先公开可核验研究；无法核实时用患者池与治疗渗透率做透明的情景分析，不伪造单点预测。

外部检索优先使用官方站点和原始文件。记录页面/文件发布日期与访问日期。

### 4.1 禁止静默降级

- InnoSight 未启用或无权限：停止专用完整版，明确建议使用通用版。
- 单一接口暂时失败：最多进行两次有依据的重试；仍失败则记录接口、错误类型、已尝试动作和受影响章节。
- 核心接口缺失：使用官方外部来源补缺，但必须在执行回执和报告口径中标记 InnoSight 覆盖缺口。
- 不得在未执行接口发现和 schema 读取的情况下，以网页搜索结果冒充 InnoSight 专用调研。

## 5. 证据记录与冲突处理

- 为每项关键事实记录：主张、数值/状态、适用地区、人群/亚型、数据日期、来源名称和链接或唯一标识。
- 对阶段冲突，优先采用日期更新的一手监管或试验记录，并说明不同来源的时间差。
- 对临床数字冲突，核对分析集、数据截止日、剂量队列、治疗线次、随访长度和终点定义，不直接取更高数字。
- 对流行病学冲突，解释病例定义、年龄范围、年份、地域和模型方法差异。
- 对交易金额冲突，优先采用交易双方公告或监管申报；不要将首付款、股权投资和潜在里程碑重复相加。

## 6. 研究执行回执

正式报告前输出以下字段：

`数据域 | InnoSight API/path | schema读取状态 | 查询词/过滤条件 | 分页终止依据 | 页数/cursor批次 | 原始结果量 | 去重结果量 | 外部核验 | 缺口`

回执必须能让复核者判断：连接器是否真正启用、是否先读 schema、是否完成翻页、是否做了同义词/亚型补检，以及结论的覆盖边界。不得披露凭证、token 或账户敏感信息。
# The Golden Schema：五部分报告结构

## 目录

1. 报告前置信息
2. 第一部分：调研的目的和意义
3. 第二部分：创新药疾病领域的背景
4. 第三部分：目前的竞争格局
5. 第四部分：BD交易情况
6. 第五部分：技术展望

## 1. 报告前置信息

标题：`创新药【疾病名称】领域综述报告`

在正文前用简短“研究口径”说明：

- 数据截止日
- 全球/中国范围
- 疾病定义、亚型及纳入/排除范围
- “创新药”“中国本土企业”“主要交易”的定义
- 关键数据源与公开信息局限

## 2. 第一部分：调研的目的和意义

从患者结局和未满足临床需求出发，简练说明：

- 疾病负担及对患者、医疗系统和社会的影响
- 现行治疗的关键缺口
- 对研发立项、BD标的筛选、竞争情报及IP监控的决策价值

避免在本部分重复完整背景，通常用2至4段完成。

## 3. 第二部分：创新药【疾病名称】的背景

### 3.1 疾病概述与病因/病理机制

- 疾病定义、主要亚型、自然病程、危险因素和病因
- 关键病理生物学机制及其与药物研发的联系
- 必要时说明诊断标准、生物标志物和患者分层

### 3.2 流行病学与趋势

分别报告全球和中国：

- 发病率、患病率、患者人数、死亡/致残负担
- 年龄、性别、地域或关键亚型分布
- 历史趋势与未来变化驱动因素

每个数字注明年份、地域、病例定义和来源。不要混用总人口患病人数、确诊人数和接受治疗人数。

### 3.3 美国与中国治疗指南

- 说明最新版指南及其发布组织/年份
