# 疾病领域调研

## 输入与默认范围

- 将疾病名称作为唯一必填输入。若缺失，只追问具体疾病领域。
- 默认覆盖全球与中国，聚焦创新治疗药物；数据截止日为执行当天，BD 与并购回溯最近 24 个月。
- 对宽泛疾病建立合理亚型分层并披露纳入/排除边界；除非无法识别疾病，否则不追加澄清问题。
- 默认以中文、专业、严谨、客观的医药行业分析口吻撰写。

## 必须读取的参考文件

执行前完整读取：

1. `references/source-and-research-protocol.md`：数据源、连接器发现、分页和证据规则。
2. `references/golden-schema.md`：固定五部分报告结构。
3. `references/quality-checklist.md`：交付前检查。

## 执行流程

1. 规范疾病中文名、英文名、缩写、旧称、关键亚型和检索同义词，确定边界。
2. 发现当前智能体可用的搜索、浏览、数据库、试验、监管、论文和专利能力；先读 schema/帮助，再调用，禁止猜测工具名或参数。
3. 先获取疾病负担、病理机制、指南与现行标准疗法，再形成未满足需求清单。
4. 建立全球管线母表，按靶点/机制和 modality 双维度分类；核对全球最高阶段、中国阶段、试验状态、关键读出与安全性。
5. 从母表中单独识别中国本土创新企业及资产，补充来源、注册开发阶段、差异化与商业化状态。
6. 系统检索最近 24 个月与该疾病治疗直接相关的许可、合作、选择权、资产收购和公司并购；记录已披露和未披露金额。
7. 严格按 Golden Schema 撰写五个一级部分，不增加平行的第六部分。
8. 完成质量检查后交付。

## 判断原则

- 明确区分已核验事实、来源间推断、分析判断和未知事项。
- 对关键数字、阶段、监管状态、临床结果与交易金额给出可追溯来源和日期。
- 不将新闻稿探索性结果写成确证性结论，不对不可比的跨试验数据作简单优劣排序。
- 使用疾病适配的疗效指标；不要把肿瘤指标机械套用于非肿瘤疾病。
- 未披露、未检索到或来源冲突的信息必须明确标注，不猜测。
- “所有交易”仅指所披露检索口径内的系统性覆盖；注明截止日、纳入标准与公开信息局限。
- 有商业数据库时用其提高覆盖率，但关键结论仍尽量回到监管、试验注册、论文或交易双方公告核验。
- 无商业数据库时继续使用权威公开来源，不得因工具缺失而伪造完整性。

## 输出要求

- 标题为“创新药【疾病名称】领域综述报告”。
- 开头注明数据截止日、地理范围、疾病/亚型范围、关键定义和公开信息局限。
- 正文只保留五个一级部分：调研的目的和意义、背景、目前的竞争格局、BD交易情况、技术展望。
- 管线、临床数据和交易优先使用结构化表格，表后给出综合判断。
- 正文就近引用，结尾列出主要来源；读者应能定位每项关键主张的证据。
# 数据源与调研协议

## 1. 能力发现与数据源优先级

先识别当前环境实际可用的能力，不假设某个品牌、工具名或字段必然存在。

优先级如下：

1. 可用的结构化医药数据连接器及其证据来源，例如 InnoSight、Citeline、Cortellis、Evaluate、Pharmaprojects、PatSnap 或机构内部数据服务。
2. 一手权威来源：FDA、NMPA/CDE、EMA、WHO、国家卫健委、CDC、ClinicalTrials.gov、WHO ICTRP、ChiCTR、指南机构、公司监管申报与交易双方公告。
3. 同行评议论文、正式会议摘要/演示、疾病登记研究与高质量系统综述。
4. 可信行业媒体、公司新闻稿和投资者材料，用于补充近期进展；关键结论尽量交叉核验。
5. 市场研究报告，仅用于市场规模参考，并注明基准年、预测期、地域、口径及实际值/预测值。

不得把搜索摘要、未经核验的聚合页或模型常识作为关键数字的唯一证据。

## 2. 结构化连接器协议

若存在结构化医药连接器：

1. 先调用其能力列表、帮助或 schema 接口，确认账户可用数据集。
2. 查询前读取准确字段、枚举、返回结构和翻页方式；禁止猜测参数。
3. 优先使用疾病流行病学、指南/治疗格局、已获批适应症、全球管线、临床试验设计/时间线/结果、中国注册、BD交易、新闻和专利等专用数据集。
4. 对管线、试验和交易持续翻页至结束；根据连接器返回的 cursor、has_more、total、page/page_size 等真实字段判断。
5. 保存疾病词、同义词、亚型、地区、阶段、交易日期范围、检索日期与去重规则。

若连接器不可用、未授权、覆盖不足或数据过旧，转向公开一手来源，不中止任务。说明降级路径和覆盖限制。

### InnoSight 适配

若当前环境提供 InnoSight MCP，可按以下顺序适配：

1. 列出可用 data APIs。
2. 对目标 API 先读取 schema。
3. 优先调用专用接口，再使用通用 query 接口。
4. 对结果完整翻页。

可重点寻找与 epidemiology、treatment landscape/guidelines、approved indications、global pipeline、clinical design/timeline/results、domestic registration、deals、news、pipeline patents 对应的接口。具体名称以实时 schema 为准，不把示例名称当作保证存在的工具。

## 3. 分主题检索

### 背景、流行病学与指南

- 同时检索疾病总称、主要亚型与旧称。
- 获取美国与中国最新版指南，必要时补充欧洲或国际共识。
- 记录指南组织、版本/年份、患者分层、治疗线次和推荐强度。
- 流行病学分别报告全球与中国，区分患病率、发病率、患者人数、确诊人数、治疗人数和预测人数。

### 全球管线与临床证据

- 先建立疾病相关完整管线，再按别名、盐型、合作关系、靶点、modality、阶段和地区归一去重。
- 分开报告全球最高阶段、中国阶段、试验获批和实际启动。
- 对已上市、关键/注册性试验、Phase 2/3、有重要读出或机制差异的资产继续核对设计和结果。
- 记录人群、样本量、分期、对照、随访、主要终点、关键疗效、安全性、数据截止日和成熟度。

### 中国本土布局

- 默认将总部或主要创新研发主体位于中国的企业视为中国本土企业。
- 区分自主发现、引进、共同开发和中国区域权益。
- 核对临床前、IND 获批、Phase 1/2/3、NDA/BLA、获批上市与商业化状态。
- 从机制、分子设计、给药、患者选择、疗效、安全性、便利性、成本与全球开发能力判断差异化。

### BD 与并购

- 覆盖截止日前 24 个月；稍早但决定当前格局的交易可作为例外并解释。
- 纳入与疾病治疗资产直接相关的许可、合作研发、共同商业化、选择权、资产收购和公司并购。
- 记录日期、双方、资产/平台、权益范围、地区、首付款、股权/近期付款、里程碑、潜在总额、特许权使用费与状态。
- 保留公告原币种；严格区分总对价、首付款、股权投资、潜在最高金额与未披露。

## 4. 证据记录与冲突处理

- 为关键事实记录主张、数值/状态、地区、人群/亚型、数据日期、来源名称、链接或唯一标识。
- 阶段冲突优先采用更新的一手监管或试验记录，并解释时间差。
- 临床数字冲突时核对分析集、数据截止日、剂量队列、线次、随访与终点定义。
- 流行病学冲突时解释病例定义、年龄、年份、地域与模型方法。
- 交易金额冲突优先交易双方公告或监管申报，不重复加总首付款、股权和里程碑。
# Golden Schema：五部分报告结构

## 报告前置信息

标题：`创新药【疾病名称】领域综述报告`

正文前说明数据截止日、全球/中国范围、疾病及亚型边界、“创新药”“中国本土企业”“主要交易”的定义、关键数据源与公开信息局限。

# 第一部分：调研的目的和意义

以 2–4 段说明疾病负担、现行治疗缺口，以及对研发立项、BD 标的筛选、竞争情报和 IP 监控的决策价值。不要重复完整背景。

# 第二部分：创新药【疾病名称】的背景

## 2.1 疾病概述与病因/病理机制

覆盖定义、主要亚型、自然病程、风险因素、病因、关键病理生物学、诊断、生物标志物和患者分层。

## 2.2 流行病学与趋势

分别报告全球与中国的发病率、患病率、患者人数、死亡/致残负担及趋势。每个数字注明年份、地域、病例定义和来源，不混用患病、确诊、治疗和市场患者池。

## 2.3 美国与中国治疗指南

说明最新版指南、组织与年份；按严重度、分期、分子亚型或线次总结路径并解释中美差异。

## 2.4 当前标准疗法

按患者分层或线次综述药物和非药物治疗。推荐表头：

`患者分层/线次 | 标准方案 | 关键依据 | 主要获益 | 主要局限 | 美国/中国差异`

## 2.5 未满足临床需求

区分疗效/耐药/复发、安全性、特定人群、诊断与 biomarker、给药负担、依从性、可及性和成本。

# 第三部分：目前的竞争格局

## 3.1 全球管线分析

先量化资产数、活跃资产数、阶段、机制、modality 和主要企业，并说明去重口径。

全球总览推荐表头：

`机制/靶点 | 药物/代号 | Modality | 原研/合作企业 | 全球最高阶段 | 中国阶段 | 目标人群/线次 | 关键试验 | 最新状态及日期`

临床证据推荐表头：

`药物 | 试验/分期 | 人群与样本量 | 主要终点 | 关键疗效 | 主要安全性 | 数据截止日 | 来源`

对每个方向判断生物学与临床验证、临床意义、数据成熟度、安全性/开发风险、拥挤度、差异化和近期催化剂。

## 3.2 中国本土创新药企布局

推荐表头：

`企业 | 药物/代号 | 来源（自主/引进/合作） | 靶点/机制 | Modality | 中国阶段 | 全球阶段 | 差异化 | 近期里程碑`

分析同质化、潜在 FIC/BIC 或本地可及性优势，以及开发、数据、知识产权和商业化风险。不得仅凭企业自称授予 FIC/BIC。

# 第四部分：BD交易情况

覆盖最近 24 个月，至少包含：

`公告日期 | 许可方/卖方 | 被许可方/买方 | 类型 | 标的及机制 | 权益地区 | 首付款 | 里程碑/潜在总额 | 特许权使用费 | 战略意义 | 来源`

金额未披露写“未披露”；“up to”保留限定。表后总结交易集中机制、modality、阶段和地域，估值与首付款信号，中国企业角色变化，以及对竞争、IP 与合作标的的启示。

# 第五部分：技术展望

基于未满足需求、竞争和交易证据，提出有条件的前瞻判断：

- 可能改变标准疗法的机制与产品特征。
- 联合治疗、使用线次与患者筛选方向。
- 分子设计、递送、给药、安全窗、biomarker 和试验设计趋势。
- 未来 2–5 年临床、监管与商业催化剂。
- 基于患者池、可治疗人群、渗透率、疗程、定价和竞争约束的透明市场情景。
- 对立项、BD 与 IP 监控的可执行建议及风险触发器。

对重大判断说明成立条件、反证因素和后续监测信号，避免无条件预测。
# 交付前质量检查

## 范围与时效

- [ ] 已披露疾病、亚型、地域、创新药和中国本土企业口径。
- [ ] 已注明执行当天的数据截止日。
- [ ] 临床、审批与交易状态已用近期来源复核。
- [ ] BD 交易覆盖最近 24 个月，例外项已解释。

## 数据与连接器

- [ ] 已先发现可用工具与 schema，未猜测工具名、字段或参数。
- [ ] 若使用结构化连接器，已完整翻页并保存检索口径。
- [ ] 若连接器不可用，已使用公开一手来源降级，并披露覆盖限制。
- [ ] 管线已按别名、合作方和地区权益归一去重。
- [ ] 全球阶段、中国阶段、试验获批与实际启动已区分。

## 临床与科学

- [ ] 关键临床数据包含人群、样本量、终点、截止日和来源。
- [ ] 疗效指标适配疾病，未机械套用肿瘤指标。
- [ ] 单臂或早期试验未写成确证性比较证据。
- [ ] 跨试验比较已提示人群、设计和随访差异。
- [ ] 安全性包含关键严重事件与停药信息，不只写“耐受性良好”。

## 流行病学、市场与交易

- [ ] 流行病学注明年份、地域、病例定义和来源。
- [ ] 患病、确诊、治疗、可治疗人群和市场患者池未混用。
- [ ] 市场规模区分历史实际、第三方预测和自主情景。
- [ ] 交易金额区分首付款、股权、里程碑、潜在总额和 royalty。
- [ ] 未披露信息未被猜测或重复加总。

## 引用与结论

- [ ] 关键数字、阶段、疗效、安全性、指南和交易均有就近引用。
- [ ] 一手来源优先，新闻稿/二手来源已尽量交叉核验。
- [ ] 来源冲突已解释。
- [ ] 事实、推断、判断和未知清晰分开。
- [ ] 正文严格为五个一级部分。
- [ ] 建议对立项、BD 与 IP 监控有明确行动指向，且未冒充正式 FTO 或法律意见。
