# 数据源与调研协议

## 1. 能力发现与数据源优先级

先识别当前环境实际可用的能力，不假设某个品牌、工具名或字段必然存在。

优先级如下：

1. 可用的结构化医药数据连接器及其证据来源，例如 InnoSight、Citeline、Cortellis、Evaluate、Pharmaprojects、PatSnap 或机构内部数据服务。
2. 一手权威来源：FDA、NMPA/CDE、EMA、WHO、国家卫健委、CDC、ClinicalTrials.gov、WHO ICTRP、ChiCTR、指南机构、公司监管申报与交易双方公告。
3. 同行评议论文、正式会议摘要/演示、疾病登记研究与高质量系统综述。
4. 可信行业媒体、公司新闻稿和投资者材料，用于补充近期进展；关键结论尽量交叉核验。
5. 市场研究报告，仅用于市场规模参考，并注明基准年、预测期、地域、口径及实际值/预测值。

不得把搜索摘要、未经核验的聚合页或模型常识作为关键数字的唯一证据。

## 2. 结构化连接器协议

若存在结构化医药连接器：

1. 先调用其能力列表、帮助或 schema 接口，确认账户可用数据集。
2. 查询前读取准确字段、枚举、返回结构和翻页方式；禁止猜测参数。
3. 优先使用疾病流行病学、指南/治疗格局、已获批适应症、全球管线、临床试验设计/时间线/结果、中国注册、BD交易、新闻和专利等专用数据集。
4. 对管线、试验和交易持续翻页至结束；根据连接器返回的 cursor、has_more、total、page/page_size 等真实字段判断。
5. 保存疾病词、同义词、亚型、地区、阶段、交易日期范围、检索日期与去重规则。

若连接器不可用、未授权、覆盖不足或数据过旧，转向公开一手来源，不中止任务。说明降级路径和覆盖限制。

### InnoSight 适配

若当前环境提供 InnoSight MCP，可按以下顺序适配：

1. 列出可用 data APIs。
2. 对目标 API 先读取 schema。
3. 优先调用专用接口，再使用通用 query 接口。
4. 对结果完整翻页。

可重点寻找与 epidemiology、treatment landscape/guidelines、approved indications、global pipeline、clinical design/timeline/results、domestic registration、deals、news、pipeline patents 对应的接口。具体名称以实时 schema 为准，不把示例名称当作保证存在的工具。

## 3. 分主题检索

### 背景、流行病学与指南

- 同时检索疾病总称、主要亚型与旧称。
- 获取美国与中国最新版指南，必要时补充欧洲或国际共识。
- 记录指南组织、版本/年份、患者分层、治疗线次和推荐强度。
- 流行病学分别报告全球与中国，区分患病率、发病率、患者人数、确诊人数、治疗人数和预测人数。

### 全球管线与临床证据

- 先建立疾病相关完整管线，再按别名、盐型、合作关系、靶点、modality、阶段和地区归一去重。
- 分开报告全球最高阶段、中国阶段、试验获批和实际启动。
- 对已上市、关键/注册性试验、Phase 2/3、有重要读出或机制差异的资产继续核对设计和结果。
- 记录人群、样本量、分期、对照、随访、主要终点、关键疗效、安全性、数据截止日和成熟度。

### 中国本土布局

- 默认将总部或主要创新研发主体位于中国的企业视为中国本土企业。
- 区分自主发现、引进、共同开发和中国区域权益。
- 核对临床前、IND 获批、Phase 1/2/3、NDA/BLA、获批上市与商业化状态。
- 从机制、分子设计、给药、患者选择、疗效、安全性、便利性、成本与全球开发能力判断差异化。

### BD 与并购

- 覆盖截止日前 24 个月；稍早但决定当前格局的交易可作为例外并解释。
- 纳入与疾病治疗资产直接相关的许可、合作研发、共同商业化、选择权、资产收购和公司并购。
- 记录日期、双方、资产/平台、权益范围、地区、首付款、股权/近期付款、里程碑、潜在总额、特许权使用费与状态。
- 保留公告原币种；严格区分总对价、首付款、股权投资、潜在最高金额与未披露。

## 4. 证据记录与冲突处理

- 为关键事实记录主张、数值/状态、地区、人群/亚型、数据日期、来源名称、链接或唯一标识。
- 阶段冲突优先采用更新的一手监管或试验记录，并解释时间差。
- 临床数字冲突时核对分析集、数据截止日、剂量队列、线次、随访与终点定义。
- 流行病学冲突时解释病例定义、年龄、年份、地域与模型方法。
- 交易金额冲突优先交易双方公告或监管申报，不重复加总首付款、股权和里程碑。
