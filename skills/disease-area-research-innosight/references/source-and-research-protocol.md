## 1. 数据源优先级

按以下顺序获取和核验信息：

1. InnoSight MCP 的结构化医药数据及其证据来源。
2. 一手权威来源：FDA、NMPA/CDE、EMA、WHO、国家卫健委、CDC、ClinicalTrials.gov、WHO ICTRP、ChiCTR、指南发布机构、公司监管申报文件和交易双方公告。
3. 同行评议论文、正式学术会议摘要/演示、疾病登记研究和高质量系统综述。
4. 可信行业媒体、公司新闻稿和投资者材料，用于补足最新进展；关键结论尽量交叉核验。
5. 市场研究报告仅用于市场规模参考，必须注明基准年、预测期、地域、口径及实际值/预测值。

不得以搜索结果摘要、未经核验的聚合页面或模型常识作为关键数字的唯一依据。

## 2. InnoSight 必须执行的调用流程

1. 首先调用 `mcp__innosight_mcp__list_data_apis`，确认当前账户可用接口。
2. 在调用任一数据查询接口之前，先以其 API path 调用 `mcp__innosight_mcp__get_data_api_schema`，读取准确的 `searchParams` 字段、枚举、返回结构和翻页方式。禁止猜测字段。
3. 优先调用专用接口；仅在专用接口不适用时使用 `mcp__innosight_mcp__query_data_api`。
4. 对需要完整覆盖的管线、临床试验、交易等结果持续翻页。依据 `pagination.has_more`、`next_cursor`、`totalCount`、`pageNum` 和 `pageSize` 判断；不得只取第一页。
5. 保存检索口径：疾病词、同义词、亚型、地区、阶段、交易日期范围和检索日期。

### 2.1 完整性门槛

- 至少尝试覆盖：流行病学、指南/治疗格局、已获批药物、全球管线、全球临床试验、临床设计/时间线、试验结果、中国注册、BD交易和近期新闻。
- 若竞争或 IP 判断是核心任务，补充 `search_global_pipeline_patent`；该结果仅作专利背景，不代替正式 FTO。
- 每个计划使用的接口均先读取 schema，并在执行回执中记录实际 API path。
- 集合型查询持续翻页，直到 `pagination.has_more=false`、`next_cursor` 为空、返回量达到 `totalCount`，或接口明确返回末页。
- 若同一接口按疾病总称无法覆盖主要亚型，使用同义词、旧称和亚型分别补检，再统一去重。
- 不得因为首批结果看起来“足够”而提前终止。

优先评估的专用接口包括：

- `search_epidemiology`：患病率、发病率、患者人数、疾病负担与趋势。
- `search_therapeutic_landscape_guide_recs`：指南推荐和治疗格局。
- `search_pipeline_approved_indication`：已获批药物及地区布局。
- `search_global_pipeline`：全球管线、靶点、药物类型、企业和阶段。
- `search_global_clinical`、`search_global_clinical_core_design`、`search_global_clinical_timeline`：试验状态、设计和时间线。
- `search_trial_result`：临床结果、终点、会议/文献证据和安全性。
- `search_domestic_registration`：中国注册与审评审批进度。
- `search_deals`：授权、合作、并购与交易金额。
- `search_pharmaceutical_news`：近期进展补充与时效性核查。
- `search_global_pipeline_patent`：仅在竞争判断需要时补充专利背景，不替代正式FTO分析。

## 3. 分主题检索路径

### 疾病背景与指南

- 同时检索疾病总称、主要亚型和旧称。
- 优先获取美国与中国最新版指南；必要时补充欧洲或国际共识。
- 记录指南名称、发布组织、版本/年份、患者分层、治疗线次和推荐强度。
- 流行病学分别报告全球与中国；区分患病率、发病率、诊断人数、治疗人数和预测人数。

### 全球管线与临床证据

- 先拉取疾病相关完整管线，再按靶点/机制、Modality、阶段、地区和企业去重。
- 同一资产的不同代号、盐型或合作方记录归一化，但保留地区权利和联合开发关系。
- 对代表性资产继续查询试验设计、关键时间线和结果。优先选择已上市、注册性/关键试验、Phase 2/3、有重要临床读出或具有显著机制差异的资产。
- 记录试验人群、样本量、分期、对照、随访、主要终点、关键疗效、安全性和数据成熟度。
- 分开报告全球最高阶段与中国阶段，不把“获准开展临床”写成“进入临床”。

### 中国本土布局

- 默认将中国本土企业定义为总部或主要创新研发主体位于中国的企业。
- 区分自主发现、License-in、共同开发和中国区域权益。
- 核对临床前、IND获批、Phase 1/2/3、NDA/BLA、获批上市和商业化状态。
- 差异化从机制、分子设计、给药方式、患者选择、疗效、安全性、便利性、成本和全球开发能力判断。

### BD交易与并购

- 检索区间为截止日前24个月，必要时纳入稍早但对当前格局有决定性影响的交易，并说明例外原因。
- 纳入与疾病治疗资产直接相关的 License-in/out、合作研发、共同商业化、选择权、资产收购和公司并购。
- 记录公告日期、交易双方、资产/平台、权益范围、地区、首付款、近期付款、研发/注册/销售里程碑、潜在总额、特许权使用费和交易状态。
- 保留公告原币种；只有在确有必要时换算，并注明汇率日期。严格区分“总对价”“潜在最高金额”和“未披露”。

## 4. 外部补缺策略

仅在 InnoSight 未覆盖、证据过旧或需要一手核验时使用外部来源：

- 指南：发布学会/协会官网及正式期刊版本。
- 流行病学：WHO、GBD、国家统计/卫生机构、疾病登记和高质量人群研究。
- 临床：试验注册、论文、会议官网、监管审评文件和申办方公告。
- 上市与审批：监管机构数据库和正式标签。
- BD：交易双方公告、监管申报、年报/季报和可信财经媒体。
- 市场规模：优先公开可核验研究；无法核实时用患者池与治疗渗透率做透明的情景分析，不伪造单点预测。

外部检索优先使用官方站点和原始文件。记录页面/文件发布日期与访问日期。

### 4.1 禁止静默降级

- InnoSight 未启用或无权限：停止专用完整版，明确建议使用通用版。
- 单一接口暂时失败：最多进行两次有依据的重试；仍失败则记录接口、错误类型、已尝试动作和受影响章节。
- 核心接口缺失：使用官方外部来源补缺，但必须在执行回执和报告口径中标记 InnoSight 覆盖缺口。
- 不得在未执行接口发现和 schema 读取的情况下，以网页搜索结果冒充 InnoSight 专用调研。

## 5. 证据记录与冲突处理

- 为每项关键事实记录：主张、数值/状态、适用地区、人群/亚型、数据日期、来源名称和链接或唯一标识。
- 对阶段冲突，优先采用日期更新的一手监管或试验记录，并说明不同来源的时间差。
- 对临床数字冲突，核对分析集、数据截止日、剂量队列、治疗线次、随访长度和终点定义，不直接取更高数字。
- 对流行病学冲突，解释病例定义、年龄范围、年份、地域和模型方法差异。
- 对交易金额冲突，优先采用交易双方公告或监管申报；不要将首付款、股权投资和潜在里程碑重复相加。

## 6. 研究执行回执

正式报告前输出以下字段：

`数据域 | InnoSight API/path | schema读取状态 | 查询词/过滤条件 | 分页终止依据 | 页数/cursor批次 | 原始结果量 | 去重结果量 | 外部核验 | 缺口`

回执必须能让复核者判断：连接器是否真正启用、是否先读 schema、是否完成翻页、是否做了同义词/亚型补检，以及结论的覆盖边界。不得披露凭证、token 或账户敏感信息。
